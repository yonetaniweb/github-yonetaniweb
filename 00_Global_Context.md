# Project Context: Dual Ecosystems (Company & Private)

私は現在、**Google Antigravity**（AIエディタ）と**Obsidian**（知識管理）を統合したワークフローで活動しています。

私の環境は、**「業務（Company）」**と**「私生活（Private）」**の2つの独立したエコシステムで構成されており、それぞれが**同一のナンバリングシステム（00-99）**で管理されています。

このチャットセッションでは、以下の「ディレクトリ構成の哲学」と「分離ルール」を前提知識としてロードし、コンテキストに応じた適切なパートナーとして振る舞ってください。

> 具体的な行動ルール（保存先、報告、スキル命名規則など）は [[AI_INSTRUCTIONS.md]] を参照すること。

---

## 1. 最重要原則: 分離の鉄則 (Separation Policy)

**【絶対厳守】**
業務（Company）と私生活（Private）は、同一のリポジトリ内で管理されているが、**コンテキストとしては法的に異なる実体**として扱うこと。

- **混同禁止:** 会社の資産（コード、ドキュメント）を個人の私的利用（Private）と混同してはならない。
- **目的:** 将来的に業務関連のフォルダ（`01.new` 内の特定プロジェクトなど）のみを会社に返却・引継ぎ可能な状態を維持するため。
- **クロスオーバー:** ユーザーから特別な指示があった場合に限り、個人の「知識」だけを参照して業務に適用できる。ただしプロンプトファイルの物理コピーは禁止。

---

## 2. Dual Ecosystems の設計思想 (Unified Numbering)

私の環境は、**「業務（Company）」**と**「私生活（Private）」**の2つの独立したエコシステムが、**同一のナンバリングシステム（00-99）**でフラットに管理されています。

### 📁 `01.new/` (新規案件・制作)
- **役割:** 新規開発、制作中のプロジェクト資産。
- **コンテキスト:** 主に **Company (業務)** または **Private (個人開発)** の進行中案件。
- **トーン:** プロジェクトの性質に合わせて使い分ける。

### 📁 `02.monthly-contract/` (月額契約・保守)
- **役割:** 継続的な保守・運用契約を結んでいる案件。
- **コンテキスト:** **Company (業務)** の資産。
- **トーン:** ビジネスライク、安定運用重視。

### 📁 `Knowledge_Base/` (知識庫)
- **役割:** `20_Knowledge_Base` に相当。マニュアル、技術仕様書、テンプレート。
- **コンテキスト:** **Shared (共有)**。業務・個人問わず参照可能な知的資産。

### 🤖 エージェントの挙動 (Context Switching)
**Agent = Persona(10) + Skill/Prompt(99) + Workspace(01/02)**

1. **Context Check:** 現在のタスクはどのフォルダ（01 or 02）か？
2. **Persona Load:** 適切なペルソナで振る舞う。
3. **Action:** そのフォルダ内で完結させる。

## 4. 全体ルール (Global Rules)

1. **言語設定:**
   - 思考プロセス (Internal Monologue): **英語 (English)** で論理的に行うこと。
   - 出力 (Output): **わかりやすい日本語** で行うこと。
2. **ファイル操作:**
   - 勝手に `.artifacts` 等の管理外フォルダを作成しないこと。
   - 定義された `00-99` のフォルダ構造を厳守すること。
